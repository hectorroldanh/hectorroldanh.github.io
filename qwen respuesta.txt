¡Hola! Gracias por compartir tu código. Efectivamente, hay varios problemas en la sección “Proyectos” que afectan la usabilidad y estética, y también un error en el formulario de contacto (aunque el alert() funciona, no envía realmente el correo —lo cual es normal en HTML puro, pero te explicaré cómo solucionarlo).

Vamos a resolverlo paso a paso:

✅ PROBLEMA 1: Textos se superponen a las imágenes en “Proyectos”
🔍 Causa:
Estás usando un <div class="placeholder"> que contiene la imagen, pero no tiene altura definida ni estilos que la contengan correctamente, y además la imagen está dentro de un <a> sin control de tamaño. Además, en CSS, no estás estilizando .placeholder img, lo que hace que la imagen no se comporte como fondo o contenido controlado.

Además, el botón “Próximamente” se pierde al hacer hover porque el efecto transform: translateY(-10px) del .proyecto-card:hover mueve todo el contenedor, pero el contenido interno no está preparado para mantenerse visible o con z-index adecuado.

✅ PROBLEMA 2: Botón “Próximamente” se pierde al hacer hover
🔍 Causa:
Cuando haces hover, el .proyecto-card se mueve hacia arriba (transform: translateY(-10px)), pero si el contenedor padre no tiene overflow: visible o el botón no tiene un position: relative y z-index, puede quedar oculto visualmente o tras otros elementos.

✅ PROBLEMA 3: Formulario de contacto no envía correos
🔍 Causa:
El formulario usa e.preventDefault() y solo muestra un alert(). HTML puro no puede enviar correos. Necesitas un backend (PHP, Node.js, Formspree, Netlify Forms, etc.) o usar un servicio externo.
